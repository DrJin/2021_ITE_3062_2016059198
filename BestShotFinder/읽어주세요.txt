1. 주의사항
 본 프로그램은 한양대학교 HCI 수업의 User Study를 위하여 제작된 프로그램입니다. 해당 프로그램을 사용하시고 다음의 설문에 반드시 응답해주시기 바랍니다.

설문지 링크 - https://forms.gle/Km7M868JwP1xjVGY6


2. 프로그램 소개
 본 프로그램은 '유사한 사진 가운데에서 원하는 갯수만큼 사진을 정리'하기 위한 목적으로 만들어졌습니다. 이를 위한 효과적인 방법을 찾기 위해 2가지의 방법을 적용하였습니다. 이 2가지 방법 중 최적의 방법을 찾는 것이 본 테스트의 목적입니다.


3. 프로그램 및 테스트 실행 방법 (실행방법.mp4를 참고하시길 바랍니다.)

1) BestShotFinder.exe 프로그램을 실행.

2) Step1. 흰색 창 우측 상단의 버튼을 클릭하여 비교할 파일들을 선택합니다.
   2-1) 실험을 위해 동봉된 4개의 test 사진 폴더 중에서 하나를 선택하여 해당 폴더의 사진들을 선택합니다.

3) Step2. 2가지의 비교 방식 중 한가지를 선택합니다.
   -Slider 방식 : 사진을 한장씩 넘겨가면서 선택하는 방식
   -Pickup 방식 : 여러장의 사진을 한번에 보면서 선택하는 방식

4) Step3. 저장 방식을 선택합니다.
   -총 몇장의 사진을 선택할 것인지 정합니다.
   -사진을 선택한 후의 처리 방법을 정합니다.
       -선택된 사진만 남기고 휴지통으로 : 선택한 사진 이외의 사진은 휴지통으로 버려집니다.
       -선택된 사진을 다음 폴더로 이동 : 선택한 사진을 하단의 선택한 폴더로 이동합니다. 선택하지 않은 사진은 기존 폴더에 남습니다.
       -선택되지 않은 사진을 다음 폴더로 이동 : 선택하지 않은 사진을 하단의 선택한 폴더로 이동합니다. 선태한 사진은 기존 폴더에 남습니다.

5) 지정한 갯수만큼의 사진을 고릅니다. 상단의 초록색 막대기가 다 차면 저장 버튼을 눌러 다음 단계로 이동합니다.

   -메뉴 설명 :저장 - 모든 사진을 다 골랐으면 실제로 시행합니다.
	   새로하기 - 1번으로 되돌아가 다시 진행합니다.
                     전체화면 - 전체화면 모드로 실행합니다.
                     끝내기 - 프로그램을 종료합니다.
 
   -Slider 방식 : 좌우 버튼을 눌러서 다른 사진으로 이동할 수 있습니다. 원하는 사진을 발견하고 클릭하면 체크 표시가 뜨고 선택이 됩니다.

   -Pickup 방식 : 모든 사진을 한번에 보면서 선택합니다. 원하는 사진은 Ctrl 키를 누르며 다중 선택이 가능합니다.

6) 저장 버튼을 누르면 4에서 지정한 방식대로 파일이 정리됩니다.
  6-1) 실험을 위하여 새로하기 버튼을 눌러 1번으로 돌아갑니다. 이전에 했던 폴더와는 다른 test 폴더를 선택해주세요.
  6-2) 3번에서 이전에 했던 방식과 다른 방식을 선택하여 실험을 반복합니다.

7) 실험을 다 마친 후 상단의 설문지 링크에 접속하여 설문을 작성해주시기 바랍니다. (Log.xml 파일이 제대로 생성되었는지 확인해주시기 바랍니다.)
